## 2. 강점 (Strengths)

### 2.1 의료법 안전 면책 (FR-68) — ⭐⭐⭐⭐⭐
- 자가진단 화면·PDF·LLM 출력 모두 면책 표기 의무
- "본 도구는 보호자 관찰 의견의 기록, 의료적 진단 아님" — 명시적 포지셔닝
- Risk #10 사전 식별 + LLM 가드레일 명시

### 2.2 후견 결정문 검증 (FR-59) — ⭐⭐⭐⭐
- 대법원 가족관계등록부 API 검증 + OCR 자동 추출 이중 안전
- 검증 실패 시 기본값 = 일반 보호자 권한 (Risk #9 대응)

### 2.3 5년 감사 로그 (FR-47, NFR-07) — ⭐⭐⭐⭐
- 모든 데이터 액션이 감사 로그 통과
- 보호자/관리자에게 접근 알림 (FR-14)
- 데이터 이동권 (FR-46) JSON 표준 — GDPR Art.20 호환

### 2.4 18세 권한 이양의 법적 정합 — ⭐⭐⭐⭐⭐
- 한국 민법 §938~§959 직접 인용 (PRD §18)
- D+30 정착 점검 + 응급 권한 잔존 (Risk #7 대응)
- 변호사 무료 상담 제휴 (FR-50)

### 2.5 AES-256 + 로컬 큐 암호화 (FR-33) — ⭐⭐⭐⭐
- 오프라인 로컬 데이터 (IndexedDB) AES-256 + 사용자 패스워드 PBKDF2
- 단말 캐시 24h 자동 삭제 (권한 회수 시)

### 2.6 ISMS-P 인증 일정 명시 — ⭐⭐⭐⭐
- M12 사전 평가 신청 + 18개월 내 정식 인증 (NFR-03·09)

## 3. 약점 / 개선점 (Weaknesses)

### 3.1 🔴 Critical — 의료법 자문 미위촉 (Risk #10 完화 미흡)
- Plan D-10에 "법률 자문 (개인정보) 미위촉" 명시
- **의료법 §27 "무자격 의료 진단 행위" 위반 위험은 자체 판단 불가** — 보건복지부 + 변호사 사전 검토 필수
- 특히 K-DST·M-CHAT-R 결과를 어린이집·소아과에 PDF 전달 시 분쟁 위험
- **권장**: M3 이전 의료법·전자상거래법 자문 위촉 + FR-68 워딩 변호사 검수

### 3.2 🔴 Critical — 개인정보 분리 동의 시퀀스 부재
- FR-15 "민감정보 분리 동의" 명시되나 **실제 동의 화면 시퀀스·UI 부재**
- 현재 가입 Step 1에 "본인인증 토큰" 안내만 있음. 의료·재산·AAC 동의 분리 흐름 미명세
- **권장**: 가입 Step 1.5 (필수) + 자녀 등록 후 추가 동의 분기 (선택) 시퀀스 명세 — 개인정보보호법 §22 의무

### 3.3 🔴 Critical — Audit Log Tamper-Proof 부재
- 5년 보존 + 보호자 알림은 명시되나 **로그 자체의 위변조 방지** (해시 체인 등) 부재
- 내부 직원에 의한 로그 조작 시나리오 미검토
- **권장**: Audit Log 해시 체인 또는 외부 타임스탬프 서비스 (KISA 공인전자서명) 통합

### 3.4 🔴 Critical — LLM 가드레일 회귀 기준 약함
- FR-67 SC-14 "의료적 표현 0건"이 분기 샘플링 수동
- LLM hallucination·prompt injection 방어 시나리오 부재
- "정식 진단이 필요합니다" 같은 회색 영역 표현 처리 명확성 부족
- **권장**: 정규식 + 임베딩 분류기 이중 가드 + LLM 출력 100% 자동 검사 (Sprint 5)

### 3.5 🟡 Important — 데이터 거주성 (Data Residency) 미명시
- AWS vs NCP 결정 보류 (Plan Q-02)
- 의료 데이터·후견 데이터는 **국내 보존 의무** (개인정보보호법 §39의2)
- **권장**: NCP 우선 검토 + Design 단계에 데이터 거주성 명시

### 3.6 🟡 Important — 침투 테스트 일정 명시
- NFR-03에 "침투테스트 연 2회" 명시되나 **MVP Beta 단계 일정 부재**
- M6 오픈 직전 또는 M6 오픈 후 침투 테스트 실시 시점 미정
- **권장**: M5 W4에 1차 침투 테스트 + M6 W4에 운영 후 재테스트

### 3.7 🟡 Important — 활동지원사 자격증 진위 검증 정책
- FR-25에 "자격증 번호 진위 검증" 명시되나 보건복지부 API 사용 가능성 미확인
- 미연계 시 시설장 수동 검증 폴백 필요
- **권장**: 영업 사전 협력 + 시설장 수동 검증 백업 흐름

### 3.8 🟡 Important — 결제 PCI DSS 컴플라이언스
- FR-54 사업자 카드 결제 — **PCI DSS Level 4 이상 준수** 필요
- 자체 카드 저장 vs PG사 토큰 사용 결정 보류
- **권장**: Toss Payments·NHN KCP 등 PG사 토큰 방식 (자체 저장 회피)

### 3.9 🟡 Important — 미성년 자녀 데이터의 부모 동의 자격
- 자녀 15세 이상 동의 의무 명시 (FR-18)
- **부모-자녀 갈등 시 자녀가 부모의 권한 위임을 거부하는 경우** 정책 부재
- 한국 민법상 미성년자의 의사 표명 능력 인정 범위 자문 필요
- **권장**: 변호사 자문 + 거부 시 보류 + 사회복지사 자문 자동 연결
