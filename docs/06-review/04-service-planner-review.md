## 2. 강점 (Strengths)

### 2.1 권한 매트릭스 5×20 = 100셀 명시 — ⭐⭐⭐⭐⭐
- 5종 사용자 × 20종 단위업무 = **모든 권한 상황을 ✅/△/❌ 3-state로 명시**
- 자가진단(B19) + AI 추이(B20)까지 일관 매핑
- "조건부 가능" (△)의 조건도 본문 상세에 기술 (예: U1 AIR-SDS 14세+ 직접 응답)

### 2.2 횡단 관심사(Cross-Cutting Concerns) 4종 분리 — ⭐⭐⭐⭐⭐
- 인증·세션 / 권한 검증 / 오프라인 큐 / 알림 매트릭스
- 모든 워크플로우가 이 4개 정책을 통과하도록 정의 → **구현 일관성 보장**

### 2.3 페르소나·FR·화면 3-way 트레이스 — ⭐⭐⭐⭐⭐
- Part B 각 단위업무에 FR-trace 명시 + 사용자별 변형 명시
- 보호자 First Value (1시간) → 7-Step 가입 → 인계서 PDF 흐름의 자연스러움

### 2.4 18세 이양 6개월 타임라인 — ⭐⭐⭐⭐⭐
- D-180 → D-150 → D-120 → D-90 → D-60 → D-30 → D-0 → D+30 8단계
- 각 단계의 시스템 행동·사용자 행동·산출물·FR 매핑

### 2.5 사용자별 First Value 명시 — ⭐⭐⭐⭐
- U2 보호자: 1시간 PDF / U3 활동지원사: 3분 인계서 수령 / U4: 케이스 1건 할당 / U5: 계정 활성화
- PRD가 보호자 외 사용자의 First Value를 약하게 다룬 부분을 워크플로우가 보강

## 3. 약점 / 개선점 (Weaknesses)

### 3.1 🔴 Critical — 권한 매트릭스 △ (조건부) 100% 명세 부재
- 100셀 중 △는 약 20개 — **각 △의 정확한 조건이 본문에 분산되어 있어 자동 검증 어려움**
- 예: B6 타임라인 조회 U3 "△ (권한 범위)" — 식사·행동만 권한 시 의료 일지 자동 마스킹은 어떻게? 텍스트 마스킹 vs 카드 숨김?
- **권장**: Design §4에 권한 정책 JSON 명세 (각 △ 조건을 정형화) + 자동 단위 테스트 100셀

### 3.2 🔴 Critical — 단위업무 간 트랜잭션 경계 미정
- B7 권한 부여 + B14 AAC 동의 + B13 알림 = 분산 트랜잭션
- 자녀 동의 (15세+) 화면에서 동의 후 보호자 알림 발사까지 **원자성** 보장 안 됨
- 실패 시 보상 트랜잭션 (Saga 패턴 등) 정의 부재
- **권장**: Design §6 트랜잭션 경계 명세 (Saga or 2PC 결정)

### 3.3 🟡 Important — 다자녀 가구 UX 명확성 부족
- 박순영(보호자)이 박지훈 + 박서연 2명을 관리하는 케이스
- 자녀 전환 UI는 명시되나 **권한 위임 시 어느 자녀에게 적용되는지** 명확성 부족
- 인계서 일괄 발행 가능 여부 (다자녀)도 미정
- **권장**: B7, B10에 "다자녀 모드" 분기 명시

### 3.4 🟡 Important — 보호자 vs 후견인 충돌 시나리오 부재
- 자녀 18세 이양 진행 중 부모가 사망·심신 미약 → 후견인 등록 시점·전환 절차 미정
- 보호자와 후견인이 동시에 권한 행사 시 우선순위 정책 부재
- **권장**: §B12 (18세 이양) + §B19 후견 결정 사이의 분기 시나리오 명세

### 3.5 🟡 Important — 활동지원사 다중 시설 근무 처리 부재
- 이수진이 푸른그룹홈 + 다른 시설 동시 근무 시 권한 관리 복잡
- 시설장 일괄 권한 갱신 (B8) 시 다른 시설 권한과 충돌 가능
- **권장**: U3 다중 소속 시나리오 명세 (별도 페르소나·테스트 케이스)

### 3.6 🟡 Important — 자가진단 결과의 외부 공유 경계 미정
- B19 K-DST 결과를 보호자가 어린이집·소아과에 PDF로 공유 가능
- **외부 공유 시 면책 문구의 강도** 명확화 필요 (현재는 페이지 상단만)
- 워터마크 적용 여부 + 인쇄·캡쳐 시에도 표시되는지 검증 부재
- **권장**: 결과 PDF의 모든 페이지 워터마크 + 인쇄·복사 시 동일 표시 보장

### 3.7 🟡 Important — 케이스 회의 자료 보호자 사전 검토 UX 명세 부재
- §B17에 "보호자 사전 검토 옵션" 언급되나 **실제 보호자 화면·알림·승인 흐름** 미정
- 24h 이내 미응답 시 자동 발행 vs 보류 정책 부재
- **권장**: Design 단계에 보호자 검토 모달 UI + 알림 시퀀스 명세

### 3.8 🟡 Important — 알림 매트릭스(PRD §9.1) ↔ Workflows 일치성 일부 약함
- PRD §9.1에 "이상 접근 탐지 → 致命/긴급" 정의되나 Workflows B13에 구체 트리거 부재
- Audit Log + 알림 발사 연결 시점 미명세
- **권장**: B13에 이상 접근 탐지 알림 트리거 명시 (FR-14 Audit + 보호자 알림)
