## 2. 강점 (Strengths)

### 2.1 5색 생애주기 시각 시스템 — ⭐⭐⭐⭐⭐
- 영유아 노랑 → 학령기 초록 → 전환기 파랑 → 성인기 보라 → 고령기 그레이
- **타임라인·페르소나 카드·자가진단 도메인 컬러까지 일관 적용**
- 색약 대응: 모든 컬러에 패턴·아이콘 동반 (NFR-05 충족)

### 2.2 인지 접근성 5원칙의 깊이 — ⭐⭐⭐⭐⭐
- 1화면 1결정 / 2단계 확인 / 실수 회복 / 주의 분산 방지 / 시간 압박 제거
- **KWCAG 2.2 AA + WCAG 2.1 Cognitive 직접 인용**
- 당사자 모드 자동 로그아웃 30분 (일반 5분의 6배)
- 부정 단어 회피 ("삭제"→"지울게요") 명시

### 2.3 당사자(U1) 모드의 별도 셸 — ⭐⭐⭐⭐⭐
- `/self/layout.tsx`에 24px 글씨 + 거대 [집]·[되돌리기] 버튼 + 보라색 헤더
- 큰 사진 + 픽토그램 + TTS 자동 재생
- AAC 동의 카드 (⭕/❌ 큰 카드 2개) — Recordare의 시그니처 UX

### 2.4 화면 명세의 일관성 — ⭐⭐⭐⭐⭐
- 41화면 모두 동일 포맷 (목적 + 화면 구성 ASCII + 콘텐츠 + UI 컴포넌트 + 메시지 톤)
- 와이어프레임 → 실제 코드 1:1 매핑 가능

### 2.5 자가진단 3화면의 의료법 안전 설계 — ⭐⭐⭐⭐⭐
- FR-68 면책 배너 모든 화면 의무 노출
- AI 추이 분석 3회 미만 시 "데이터 부족" 안내 — 오해석 방지

### 2.6 모드 토글 (일반·쉬운말·당사자) — ⭐⭐⭐⭐

- 한 번 선택하면 전 화면·전 세션 유지

## 3. 약점 / 개선점 (Weaknesses)

### 3.1 🔴 Critical — 디자인 토큰 코드화 부재 (Design Anchor)
- BI 가이드에 컬러/타이포/spacing 명시되어 있으나 **`tailwind.config.ts` 외 정식 design tokens 파일 없음**
- Figma 디자인 라이브러리 미연결 (수동 동기화 위험)
- 다크 모드·고대비 모드 검토 안 됨
- **권장**: Design 단계에 `docs/02-design/styles/recordare.design-anchor.md` 정식 캡처 (BI v1.0 토큰 + 컴포넌트 변형)

### 3.2 🟡 Important — 글자 크기 4단계 적용 일관성 약함
- 14/16/20/24px 토큰은 있으나 사용자별 기본값 (U4 14px, U1 24px 등) **저장·복원 흐름 미정**
- 다중 자녀 보호자의 자녀별 텍스트 크기 개별 조정 필요성 검토 안 됨
- **권장**: 사용자 설정 모델 (UserPreferences)에 fontSize, mode, voice 등 명시

### 3.3 🟡 Important — 다크 모드·고대비 모드 검토 부재
- WCAG 1.4.3 (대비 4.5:1) 충족하지만 야간 사용자·시력 약자에게는 다크 모드 권장
- 현재 BI 컬러는 라이트 모드 가정. 다크 변형 토큰 없음
- **권장**: Design 단계에 다크 모드 토큰 + 고대비 모드 옵션 추가 (P1)

### 3.4 🟡 Important — 다국어 확장성 미반영
- NFR-08에 "한국어 우선, 영어 v2" 명시되나 **i18n 인프라 (next-intl 등) 결정 안 됨**
- 쉬운말 모드 어휘 1,500어휘 사전 + 영어 어휘 번역 매핑 필요
- **권장**: 다국어 키 명명 규칙 + 쉬운말 변환 알고리즘 명세 (P1)

### 3.5 🟡 Important — 모바일 최적화 검토 부재
- 41화면 모두 lg:grid 기반 데스크탑 우선 레이아웃
- iOS Safari · Android Chrome 실기 검증 없음 (PWA 단일 코드베이스 가정)
- **권장**: Sprint 5에 모바일 실기 테스트 (5종 기기 × 4가지 화면 회전)

### 3.6 🟢 Minor — 페르소나 카드 아이콘 단순함
- `/parent` 5색 카드의 emoji (👶🧒👦🧑👵)는 빠른 프로토타입에 적합
- Hi-fi 단계에서 일러스트 또는 사진 교체 권장
