|------|------|
| **Problem** | 온보딩 완료 후 첫 1주 "혼자 쓰는 느낌" 페인포인트 (PRD §7.1). 발달장애 자녀를 키우는 보호자는 비슷한 상황의 가족과 연결되고 싶지만, 장애 정보가 노출될 수 있는 오픈 커뮤니티(에이블링 등)는 신뢰하기 어렵다. Recordare 내 인증된 보호자 한정 공간이 없다. |

## 1. Background

### 1.1 신규 기능 추가 배경

Recordare MVP Beta(recordare.plan.md v1.3)에서 온보딩 후 첫 1주 잔존율 70% 달성이 핵심 KPI(KPI-01)다. 현재 PRD §7.1은 "혼자 쓰는 느낌"을 온보딩 1주 페인포인트로 명시하고 있으나, 이를 해소하는 기능이 MVP Beta에 없다.

보호자 자조 모임은 다음 두 목적을 동시에 달성한다:
1. **잔존율 강화**: 비슷한 상황의 보호자와 연결 → 플랫폼 이탈 감소
2. **입소문(바이럴)**: 보호자 커뮤니티에서 Recordare 자연 홍보

### 1.2 경쟁 분석과 차별화

| 비교 | 에이블링 | Recordare 자조 모임 |
|------|---------|-------------------|
| 접근성 | 오픈 (비가입자도 열람 가능) | 폐쇄형 (인증 보호자만) |
| 그룹 구성 | 수동 선택 | 생애주기 자동 매칭 + 자유 개설 |
| 기록 연동 | 없음 | 없음 (완전 분리, 신뢰 유지) |
| 주요 가치 | 정보 공유 | 동류 연결 + 안전한 공간 |

## 5. 기술 고려사항

### 5.1 데이터 모델 (Community 전용 스키마)

```
Community (자조 모임 전용 DB 스키마 — 케어 데이터와 완전 분리)

Group
  id, name, description, type(AUTO_MATCHED|USER_CREATED)
  lifecycleStage(INFANT_EARLY|SCHOOL_AGE|TRANSITION|ADULT|SENIOR)
  region?, maxMembers, isPublic, createdBy(guardianId), createdAt

GroupMember
  groupId, guardianId, role(ADMIN|MEMBER), joinedAt, status

Post
  id, groupId, authorId(guardianId), content, images[], likeCount
  isBlinded, createdAt, updatedAt

Comment
  id, postId, authorId, content, isBlinded, createdAt

Meeting
  id, groupId, organizerId, title, description
  type(ONLINE|OFFLINE), location?, onlineUrl?
  scheduledAt, maxAttendees, rsvpType(FIRST_COME|APPROVAL)
  createdAt

MeetingRsvp
  meetingId, guardianId, status(PENDING|CONFIRMED|CANCELLED), createdAt

Report
  id, targetType(POST|COMMENT|MEETING), targetId
  reporterId, reason, status(PENDING|RESOLVED), createdAt
```

### 5.2 API 엔드포인트 (주요)

| Method | Path | 설명 |
|--------|------|------|
| GET | /api/community/groups | 그룹 목록 (자동 매칭 + 탐색) |
| POST | /api/community/groups | 자유 그룹 개설 |
| GET | /api/community/groups/:id | 그룹 상세 (게시판 + 모임) |
| POST | /api/community/groups/:id/join | 그룹 가입 |
| GET | /api/community/groups/:id/posts | 게시판 목록 |
| POST | /api/community/groups/:id/posts | 글 작성 |
| POST | /api/community/posts/:id/comments | 댓글 작성 |
| POST | /api/community/groups/:id/meetings | 모임 일정 등록 |
| POST | /api/community/meetings/:id/rsvp | 참여 신청/취소 |
| POST | /api/community/reports | 신고 접수 |

### 5.3 스택 연계

- 기존 Recordare 스택 (Next.js 15, Node/Express, PostgreSQL, Redis) 그대로 사용
- 커뮤니티 DB 스키마는 `community` Prisma 스키마로 분리
- 파일 업로드(게시물 사진): 기존 FR-26 파일 스토리지 인프라 재사용
- 알림(FR-79): 기존 알림 시스템(FR-35~38) 채널 재사용
