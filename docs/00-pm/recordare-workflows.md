## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem** | PRD에 FR 47건 + NFR이 정의되었지만, "누가 어떤 순서로 무엇을 하는가"가 흩어져 있어 Design/Do 단계에서 화면-API-권한 매핑이 어렵다. |
| **Solution** | 5종 사용자(당사자/보호자/활동지원사/사회복지사·특수교사/시설장)의 End-to-End 여정과 18종 단위업무 시퀀스를 동일 포맷(Actor → Trigger → Step → Output → FR-trace)으로 정리. |
| **Function/UX** | 화면 단위 설계가 아닌 "행동 흐름" 단위 명세 → API 계약 도출 가능, 권한 매트릭스 자동 검증, Zero Script QA 시나리오 직결. |
| **Core Value** | "PRD가 약속한 가치를 실제 동작 순서로 번역" — Plan/Design 단계의 인지 부하를 50% 이상 절감하고, 의존성 충돌(권한·오프라인·알림)을 사전 식별. |

## Context Anchor

| 차원 | 내용 |
|------|------|
| **WHY** | PRD는 "무엇을(What)"을 정의하지만 워크플로우는 "어떻게 흐르는가(How it flows)"를 정의한다. 이 둘이 다른 문서에 있어야 Design 단계에서 화면/API/권한 충돌을 조기 발견할 수 있다. |
| **WHO** | 1차 독자: Plan/Design 단계 PM·아키텍트 · 2차 독자: Do 단계 개발자(API 계약 도출) · 3차 독자: QA(테스트 시나리오 도출) |
| **RISK** | (1) 권한 모델 5종 × 업무 18종 = 90개 셀, 일부 셀의 정책이 PRD에 미정의 → Plan에서 명시 필요. (2) 오프라인/알림은 거의 모든 업무에 교차 → 횡단 관심사 분리 필요. |
| **SUCCESS** | (1) Plan 단계 진입 시 본 문서의 모든 워크플로우가 FR 번호와 1:1 매핑 (2) Design 단계에서 본 문서 기반 화면 명세 가능 (3) QA가 본 문서만으로 TC 도출 가능 |
| **SCOPE** | 포함: 5종 사용자 × 18종 단위업무. 제외: IoT/PHR 연동(v2.0 PRD 제외 범위 유지), 결제 게이트웨이 상세 흐름(Plan 단계로 위임). |

### 0.2 워크플로우 표기법

각 워크플로우는 다음 6요소로 표기한다.

| 요소 | 의미 |
|------|------|
| **Actor** | 행동의 주체 (U1~U5 코드) |
| **Trigger** | 시작 조건 (시간/이벤트/사용자 의도) |
| **Pre-condition** | 선행 조건 (로그인, 권한, 데이터 존재) |
| **Step** | 순서 있는 행동/시스템 응답 |
| **Output** | 산출물 (데이터/알림/문서/상태 변화) |
| **FR-trace** | PRD FR 번호 매핑 (검증 가능성 확보) |

### 0.3 횡단 관심사(Cross-Cutting Concerns)

모든 워크플로우는 다음 4개 횡단 정책의 영향을 받는다.

| 관심사 | 정책 |
|--------|------|
| **인증·세션** | 로그인 미상태 → 모든 액션 차단, 당사자 모드는 자동 로그아웃 30분 (NFR-09 보강) |
| **권한 검증** | 모든 데이터 액션은 (Actor, Subject, Action, Time) 4-tuple로 정책 평가 → 거부 시 감사 로그 기록 |
| **오프라인 큐** | 네트워크 단절 시 모든 쓰기 액션은 로컬 큐로 폴백(FR-30) |
| **알림 발사** | 모든 상태 변경은 알림 매트릭스(PRD §9.1)를 통과해 채널·수신자 결정 |

# Part A. 사용자별 워크플로우 (User-Centric)

각 사용자의 가입부터 활용까지 End-to-End 여정. "하루/한주/한달" 단위로 묶어 실제 사용 패턴을 반영.

## A1. 당사자 (U1) 워크플로우

### A1.1 페르소나 컨텍스트
- 17세 경계성 지적장애, AAC·쉬운말 모드 사용
- 보호자가 계정 활성화한 후 학교/가정에서 본인이 직접 일부 기능 사용
- 핵심 목표: 자기결정권 행사 (동의/거부 표명) + 일기 확인

### A1.2 First-Time 여정 (가입~첫 사용)

```
[보호자가 사전 활성화]
   ↓
[1] 보호자 단말에서 "당사자 계정 활성화" → 음성 안내 ON, 글자 크기 큼 사전 설정
[2] 학교/가정에서 태블릿 인계, 당사자 모드로 첫 로그인 (얼굴 사진 + 음성 인사)
[3] 환영 화면: "민지님, 안녕하세요" (TTS 자동 재생)
[4] 튜토리얼 3단계: ① 일기 보기 → ② 사진으로 답하기 → ③ 동의 카드 사용
[5] 첫 동의 행사 트리거 (예: 새로 만난 활동지원사 인사)
[6] 동의 완료 후 축하 애니메이션 + 보호자에게 자동 알림
```

| Step | Actor | UI | Output | FR-trace |
|------|-------|-----|--------|----------|
| 1 | U2 | 보호자 설정 메뉴 | 당사자 계정 활성화 | FR-19 (관리 책임) |
| 2 | U1 | 당사자 모드 로그인 화면 | 세션 생성 (자동 로그아웃 30분) | NFR-10 |
| 3 | 시스템 | TTS + 큰 사진 | 환영 화면 노출 | FR-17 |
| 4 | U1 | 튜토리얼 3장 | 튜토리얼 완료 플래그 | FR-17 |
| 5 | U1 | 사진+이름+픽토그램 카드 | 동의 의사 표명 | FR-18, FR-19 |
| 6 | 시스템 | 축하 화면 | 보호자에게 5분 내 푸시 | FR-19 |

### A1.3 일상 여정 (Daily Use)

| 시점 | 행동 | 화면 |
|------|------|------|
| 오전 | "오늘 일기 보기" → 어제의 사진 일지 슬라이드쇼 (활동지원사가 작성한 것) | 타임라인 뷰 (큰 카드, 5장씩) |
| 점심 | (활동지원사가 음성일지 작성) → 당사자는 "확인" 도장 찍기 (선택) | 일지 카드 + [확인했어요] 버튼 |
| 오후 | 학교 자립생활 수업 — "내가 좋아하는 활동 골라보기" → 향후 케어 계획에 반영 | 픽토그램 선택 카드 4장 |
| 저녁 | 부모가 권한 갱신 요청 시 → 당사자에게 "한 번 더 확인" 카드 | Double-confirm 모달 |

### A1.4 핵심 의사결정 여정 (18세 권한 이양)

자녀가 만 18세가 되는 시점, 보호자 → 당사자 권한 이양은 6개월간 단계적으로 진행. (이 흐름은 Part B의 B12 워크플로우와 연결.)

```
D-180일 : 보호자 알림 "민지님의 권한 이양 준비를 시작할까요?"
D-150일 : 모의 동의 시뮬레이션 (당사자가 가상의 동의 카드 3장 연습)
D-120일 : 변호사 무료 상담 예약 (제휴) — 후견 vs 자율 결정
D-90일 : 단계적 권한 이양 (의료 1개 항목부터 시작)
D-30일 : 본격 이양 — 모든 권한 메뉴에 [당사자에게 묻기] 버튼 추가
D-0일 (생일) : 권한 모델 자동 전환 — "보호자 + 당사자 공동" 또는 "당사자 단독"
```

### A1.5 당사자 UX 절대 원칙 (모든 흐름 공통)
- **1화면 1결정**, **2단계 확인 의무**, **취소 항상 노출**, **시간 압박 없음**, **부정 단어 회피** (PRD §10.2)

## A2. 보호자 (U2) 워크플로우

### A2.1 페르소나 컨텍스트
- 54세, 발달장애 자녀 박지훈(20세)의 어머니
- 30년간 종이 노트·카톡·갤러리에 산재한 기록 통합 + 자녀 자립 준비
- 핵심 목표: ① 평생 기록 통합 ② 권한 위임·회수 통제 ③ 권한 이양 안전 진행

### A2.2 First Value 여정 (가입 ~ 1시간 안에 인계서 PDF)

PRD §7의 7-Step 온보딩 마법사를 1시간 First Value 약속과 결합.

```
[Step 1] PASS/카카오 본인인증 (5분)
[Step 2] 자녀 프로필 등록 (이름/생년월일/장애 등급) (5분)
[Step 3] 현재 소속 기관 선택 (3분)
[Step 4] (선택) 기존 기록 마이그레이션 — 백그라운드로 시작 (5분 입력)
[Step 5] 권한 모델 자동 추천 (자녀 나이 기반) (3분)
[Step 6] 알림 채널 설정 (3분)
[Step 7] First Value 체험 — 미리 분류된 사진 10장 타임라인 + 인계서 샘플 PDF 다운로드 (5분)
─────────────────────────────────────────────────────
누적 30분 안에 가입 완료, 1시간 안에 PDF 다운로드 달성 KPI (FR-29)
```

| Step | Pre-condition | Output | FR-trace |
|------|---------------|--------|----------|
| 1 | 휴대폰 본인 명의 | 본인인증 토큰 | FR-25 |
| 2 | 자녀 주민번호 동의 | 자녀 프로필 ID | FR-25 |
| 3 | (선택 가능) | 기관 매핑 | FR-25 |
| 4 | (스킵 가능) | 마이그레이션 작업 ID + 푸시 예약 | FR-26, FR-27 |
| 5 | 자녀 나이 산정 완료 | 권한 모델 ENUM 저장 | FR-25 |
| 6 | (전체 OFF 불가, "중요알림 항상 ON" 강제) | 알림 설정 저장 | FR-38, Risk #5 대응 |
| 7 | 마이그레이션 완료 OR 샘플 사용 | PDF 인계서 1건 다운로드 | FR-29, FR-43 |

### A2.3 일상 여정 (Daily / Weekly)

| 주기 | 행동 | 트리거 |
|------|------|--------|
| 매일 | 자녀 일지 다이제스트 확인 (활동지원사·학교 작성분) | 21:00 푸시 1회 |
