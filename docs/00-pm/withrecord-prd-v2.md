## §6. 보완 — 누락된 사용자 시나리오

### 6.1 시나리오 A — 당사자가 직접 플랫폼을 사용하는 AAC 케이스

#### 6.1.1 페르소나 컨텍스트
- **김민지 (17세, 경계성 지적장애, Persona #1)** — v1.0 §3.2.1 참조
- **상황 가정**: 학교 자립생활 수업에서 본인 계정 첫 사용
- **선결 조건**: 보호자가 "당사자 계정 활성화" 동의, 음성 안내 ON, 글자 크기 大

#### 6.1.2 시나리오 흐름

```
[1] 학교 자립생활 수업 — 담임이 태블릿을 민지에게 건넴
   └─ 첫 화면: 큰 사진(엄마, 담임, 활동지원사 이수진) 3장
   └─ 음성: "민지님, 누가 보여요?"
   └─ 민지: "엄마요" (음성 인식) 또는 엄마 사진 탭
        ↓
[2] 환영 화면
   └─ "민지님, 오늘은 새로운 친구가 인사해요"
   └─ 사진 + 이름 + 픽토그램(직업: 활동지원사)
   └─ 음성: "이 친구가 민지님 일기를 봐도 될까요?"
        ↓
[3] AAC 동의 (Yes/No 카드)
   └─ 큰 카드 2장: [동그라미 = 좋아요] [엑스 = 싫어요]
   └─ 한 번 더 확인: "정말로 보여줘도 될까요?" (실수 방지)
        ↓
[4] 동의 완료 — 축하 애니메이션
   └─ "고마워요! 민지님이 정해줬어요"
   └─ 음성으로 "엄마에게 알려드릴게요" 안내
        ↓
[5] 보호자(엄마)에게 푸시 알림 → "민지님이 직접 동의했어요"
```

#### 6.1.3 핵심 UX 원칙 (당사자 모드 전용)
- **1화면 1결정**: 동시에 2개 이상의 선택지를 묻지 않는다
- **2단계 확인**: 모든 권한·동의 행동은 "한 번 더 묻기" 필수
- **취소는 항상 가능**: 어떤 화면에서도 큰 [집] 버튼으로 처음 화면 복귀
- **부정적 단어 회피**: "삭제" 대신 "지울게요", "오류" 대신 "다시 해볼게요"
- **시간 압박 없음**: 자동 타임아웃 없음, 본인이 스스로 닫을 때까지 유지

#### 6.1.4 신규 FR (당사자 모드)

| FR# | 요구사항 | 우선순위 |
|-----|---------|---------|
| FR-17 | 당사자 모드: 픽토그램+음성+큰글씨 3종 동시 제공 | P0 |
| FR-18 | 모든 권한 동의 화면에서 2단계 확인(Double-confirm) 의무 | P0 |
| FR-19 | 당사자 동의 행사 시 보호자에게 5분 내 푸시 알림 | P0 |
| FR-20 | "지우기/되돌리기" 기능 항상 노출 (모든 화면 좌상단) | P1 |

### 6.2 시나리오 B — 시설장이 여러 이용자를 동시 관리하는 케이스

#### 6.2.1 페르소나 컨텍스트
- **(신규 페르소나) 정원장 (52세, 그룹홈 시설장)** — 정원 12명 그룹홈 운영, 활동지원사 5명 관리
- **상황**: 분기별 인계서 정산 + 활동지원사 계약 갱신 시점 일괄 처리

#### 6.2.2 시나리오 흐름

```
[1] 시설장 대시보드 진입 (월요일 오전 9시)
   └─ KPI 카드 3개: 이용자 12명 / 활동지원사 5명 / 이번 주 만료 3건
        ↓
[2] "이번 주 만료 3건" 클릭
   └─ 활동지원사 이름·만료일·이용자 매핑 표 (D-7/D-3/D-1)
   └─ 일괄 갱신 체크박스
        ↓
[3] 일괄 갱신 클릭 → 일괄 권한 갱신 확인 모달
   └─ "5명 중 3명 / 이용자 8명에 영향"
   └─ 갱신 기간(3/6/12개월) 선택 → 일괄 적용
        ↓
[4] 인계서 정산 탭 이동
   └─ 분기 동안 생성된 인계서 7건 일괄 승인/송부
   └─ B2G 바우처 대상 인계서 자동 식별 + 청구 리포트 다운로드
        ↓
[5] 결제/구독 현황 → 익월 구독료 자동 결제 예고
```

#### 6.2.3 핵심 UX 원칙 (시설장 모드)
- **다중 선택 우선**: 모든 리스트는 체크박스 기본 노출
- **권한 영향 미리보기**: 일괄 처리 전 "N명에게 영향" 항상 명시
- **감사 로그 1-Click**: 누가/언제/무엇 했는지 CSV 즉시 내보내기
- **법령 부합 표시**: 화면 우상단에 ISMS-P 인증 배지 + 마지막 감사일

#### 6.2.4 신규 FR (시설장 모드)

| FR# | 요구사항 | 우선순위 |
|-----|---------|---------|
| FR-21 | 시설장 대시보드: 이용자/직원/계약 만료 3-KPI 카드 | P0 |
| FR-22 | 활동지원사 권한 일괄 갱신/회수 (영향 범위 사전 미리보기) | P0 |
| FR-23 | 분기 인계서 일괄 승인 + B2G 바우처 청구서 자동 생성 | P1 |

## §7. 보완 — 신규 보호자 온보딩 (First Value 1시간 안에)

### 7.1 온보딩 페인포인트 진단

| 단계 | 기존 페인 | 목표 First Value |
|------|----------|----------------|
| 가입 후 첫 5분 | "뭘 해야 할지 모르겠다" | 자녀 프로필 1개 완성 |
| 첫 30분 | "사진 5,000장을 어떻게 올리지?" | 사진 100장 자동 분류 결과 미리보기 |
| 첫 1시간 | "이게 정말 도움이 될까?" | 인계서 샘플 1장 PDF 다운로드 |
| 첫 1주 | "혼자 쓰는 느낌이다" | 학교 담임/활동지원사 1명 연결 완료 |

### 7.2 7단계 온보딩 마법사

```
[Step 1] 환영 + 본인인증 (PASS/카카오)
   └─ 진행 바: ●○○○○○○ (1/7)
        ↓
[Step 2] 자녀 등록 (이름/생년월일/장애 등급)
   └─ 자녀가 여러 명이면 "또 한 명 등록" 가능 (스킵 가능)
   └─ 진행 바: ●●○○○○○
        ↓
[Step 3] 현재 소속 기관 선택 (학교/복지관/없음)
   └─ 자동완성 + "직접 입력" 옵션
   └─ 추후 변경 가능 안내
        ↓
[Step 4] 기존 기록 마이그레이션 (선택)  ★ 핵심
   ┌─────────────────────────────────────┐
   │ 카톡 사진/영상   [업로드] (1만장까지) │
   │ 휴대폰 갤러리   [선택]              │
   │ 종이 노트       [사진 찍기→OCR]     │
   │ 엑셀/한글 파일  [드래그앤드롭]      │
   │ 건너뛰기        [나중에 할게요]     │
   └─────────────────────────────────────┘
   └─ 백그라운드 처리 + 진행률 (스킵 가능)
        ↓
[Step 5] 권한 모델 선택 (자녀 나이 기반 자동 추천)
   └─ 만 18세 미만: "보호자 전권" 자동
   └─ 만 18세 이상: "보호자 + 당사자 공동" 추천
   └─ 후견 결정문 있음: "후견 모드" 안내
        ↓
[Step 6] 알림 채널 설정 (푸시/SMS/이메일/카톡)
   └─ 권한 만료·인계서 수신·신규 기록 3종 기본 ON
        ↓
[Step 7] First Value 체험
   ├─ 미리 분류된 사진 타임라인 미리보기 (10장)
   ├─ AI 자동 생성 인계서 샘플 PDF 다운로드
   └─ "축하합니다! 위드레코드 사용 준비 완료"
```

### 7.3 마이그레이션 마법사 상세 (Step 4)

#### 7.3.1 입력 채널별 처리

| 채널 | 처리 방식 | 시간 |
|------|----------|:----:|
| 카톡 사진 (.zip 백업) | 사진 EXIF 날짜 → 시기 자동 분류 + GPT-Vision으로 식사/활동/의료 자동 태깅 | 1만장 / 14일 |
| 휴대폰 갤러리 | 동일 처리 + 위치 정보 익명화 | 5천장 / 7일 |
| 종이 노트 사진 | OCR (네이버 클로바 OCR) → 텍스트 추출 + 날짜 추정 | 100장 / 3일 |
| 엑셀/한글 | 행 단위 파싱 (날짜/이벤트/메모 컬럼 자동 매핑) | 즉시 |

#### 7.3.2 마이그레이션 중 UX 약속
- **백그라운드 처리 + 푸시 알림**: 보호자는 앱 닫고 일상 가능
- **정확도 70% 보장 + 보호자 검수 UI**: AI가 분류한 결과를 보호자가 카드 스와이프로 검증
- **되돌리기 가능**: 잘못 분류된 사진 한 번에 재분류
- **개인정보 분리 동의**: 마이그레이션 전용 별도 동의서 (얼굴 인식 사용 여부 옵트인)

## §8. 보완 — 오프라인 연속성 (Offline-First 음성 기록)

### 8.1 시나리오 — 활동지원사 이수진의 산간 캠프

```
오전 10시 : 강원도 산간 발달장애인 캠프 도착 (4G 없음)
오전 11시 : 식사 시간 음성 일지 1건 작성 → 로컬 큐 저장
오후 02시 : 산책 중 돌발행동 사건 → 음성+사진 첨부 일지 → 로컬 큐 저장
오후 06시 : 캠프 복귀, WiFi 연결됨
            → 앱이 자동으로 큐에 쌓인 5건 백그라운드 동기화
            → 충돌 감지 (동일 시간대 보호자 기록 1건 발견)
            → 사용자에게 병합/덮어쓰기 선택 UI 제공
            → 완료 후 "오늘의 일지 5건이 모두 안전하게 동기화되었습니다" 알림
```

### 8.2 기술 요구사항

| 항목 | 내용 |

| **저장 용량** | 1인당 최대 500MB (오프라인 음성 200건 ~ 2시간 분량) |
| **암호화** | 로컬 저장 시에도 AES-256 + 사용자 패스워드 기반 키 |

## §10. 보완 — 인지 접근성 (Cognitive Accessibility) 심화

### 10.1 v1.0 한계 진단

v1.0 NFR-05는 "색약/저시력"만 명시. 인지 접근성(쉬운말, 반복 확인, 실수 회복, 주의 분산 방지)은 미정의 → v2에서 보강.

### 10.2 인지 접근성 5대 원칙 (한국형 발달장애인 웹접근성 지침 + WCAG 2.1 Cognitive)

#### 원칙 1. 쉬운말 (Easy-to-Read) 모드 전역 지원

| 항목 | 기준 |
|------|------|
| 어휘 | 초등 2학년 수준 어휘 (한국어 기초 1,500어휘 내) |
| 문장 길이 | 최대 15음절 |
| 한자/외래어 | 0건 (모두 한글 풀어쓰기) |
| 픽토그램 동반 | 모든 핵심 동작 버튼은 픽토그램 + 글자 병기 |
| 음성 안내 | 모든 텍스트는 TTS로 들을 수 있음 |

**예시**:
| 일반 모드 | 쉬운말 모드 |
|----------|-----------|
| "권한 이양에 동의하시겠습니까?" | "선생님이 내 일기를 볼 수 있게 해줄까요? (그림 + 음성)" |
| "민감정보 처리 동의" | "내 비밀 이야기를 보여줄게요. 괜찮을까요?" |
| "오류가 발생했습니다" | "다시 한 번 해볼게요" |

#### 원칙 2. 반복 확인 UI (Double-Confirm)
- 모든 비가역 행동(삭제·권한 해지·동의·전송)은 2단계 확인 필수
- 1단계: 행동 카드 탭
- 2단계: "정말로 할까요? — [네/아니오]" 모달 (3초 후 자동 닫힘 없음)

#### 원칙 3. 실수 회복 (Undo) 항상 노출
- 모든 화면 좌상단 [되돌리기] 큰 버튼
- 최근 10개 행동까지 되돌리기 가능
- 영구 삭제도 30일 휴지통 보존

#### 원칙 4. 주의 분산 방지
- 동시에 떠 있는 모달/팝업 1개만 허용
- 자동재생 영상 금지, 깜박이는 광고/배너 0개
- 한 화면에 CTA(주요 행동 버튼) 1~2개만

#### 원칙 5. 시간 압박 제거
- 자동 로그아웃 시간 30분 (일반 5분의 6배) — 당사자 모드 시
- 폼 입력 자동 저장 (Draft) 매 키스트로크
- "X초 안에 답하세요" 타이머 일체 금지

### 10.3 인지 접근성 검증 도구
- **사용성 테스트**: 발달장애 당사자 자문단 5명 분기별 평가
- **자동 검사**: WCAG 2.1 AA + 한국형 웹접근성 지침 + 자체 "쉬운말 어휘 검사기"
- **인증**: 한국웹접근성평가센터 인증 + 보건복지부 정보접근성 인증

### 10.4 신규 FR/NFR (인지 접근성)

| FR/NFR# | 요구사항 | 우선순위 |
|---------|---------|---------|
| FR-40 | 쉬운말 모드 전역 토글 (어휘/문장/픽토그램/TTS 동시 적용) | P0 |
| FR-41 | 모든 비가역 행동에 2단계 확인 UI 의무 | P0 |

### 12.4 v2.0 Pre-mortem 추가 리스크

#### Risk #4. 신규 보호자가 마이그레이션 중 이탈 (확률 中 / 영향 高)
**가상**: 보호자가 카톡 사진 1만장 업로드 후 AI 분류 결과의 오류를 보고 "AI 못 믿겠다" 이탈.
**대응**: First Value를 사진 마이그레이션 이전에 배치 (Step 3까지 완주 시 인계서 샘플 즉시 제공), 분류 정확도 70% 보장 명시 + 보호자 직접 정정 UI 강화.

#### Risk #5. 알림 피로로 인한 무음 처리 → 권한 만료 사고 (확률 中 / 영향 高)
**가상**: 보호자가 일지 알림 폭주로 전체 알림 OFF → 권한 만료 D-1 알림도 미수신 → 활동지원사 데이터 접근 차단으로 입소 일정 차질.
**대응**: 알림 유형별 분리 토글 강제 (전체 OFF 옵션 없음, "중요 알림은 항상 ON" 가이드), 만료 D-1은 SMS+카톡 다중 채널 필수.

#### Risk #6. 시설장 일괄 권한 갱신 실수 (확률 中 / 영향 中)
**가상**: 시설장이 5명 활동지원사 권한을 6개월로 갱신해야 하는데 실수로 1개월 선택 → 한 달 후 일괄 만료.
**대응**: 일괄 갱신 전 "영향 범위 + 갱신 결과 만료일" 명확 미리보기, 시설장 행동 30일 내 1회 일괄 되돌리기 가능.
