### 0.1.3 자가진단·법적 안전 원칙

- K-DST, M-CHAT-R, AIR-SDS, K-SIB-R, VABS-3는 **진단서가 아니라 보호자 관찰 기록·선별·추이 관리 도구**로 표현한다.
- 모든 자가진단 화면, PDF, 외부 공유물, AI 인사이트는 FR-68에 따라 "보호자 관찰 의견 / 정식 진단은 전문의 상담" 고지를 포함한다.
- 건강·장애·자가진단·후견 정보는 개인정보보호법상 민감정보로 취급하고 FR-15 분리 동의, FR-13 암호화, FR-14 접근 알림·감사 로그를 적용한다.
- IEP/개별화교육계획, 발달재활서비스, 후견 관련 법적 근거와 공식 자료 링크는 기준 문서 §5를 원천으로 관리한다.

### 6.1 시나리오 A — 당사자가 직접 플랫폼을 사용하는 AAC 케이스

#### 6.1.1 페르소나 컨텍스트
- **김민지 (17세, 경계성 지적장애, Persona #1)** — v1.0 §3.2.1 참조
- **상황 가정**: 학교 자립생활 수업에서 본인 계정 첫 사용
- **선결 조건**: 보호자가 "당사자 계정 활성화" 동의, 음성 안내 ON, 글자 크기 큼

#### 6.1.2 시나리오 흐름

```
[1] 학교 자립생활 수업 — 담임이 태블릿을 민지에게 건넴
   └─ 첫 화면: 큰 사진(엄마, 담임, 활동지원사 이수진) 3장
   └─ 음성: "민지님, 누가 보여요?"
   └─ 민지: "엄마요" (음성 인식) 또는 엄마 사진 탭
        ↓
[2] 환영 화면
   └─ "민지님, 오늘은 새로운 친구가 인사해요"
   └─ 사진 + 이름 + 픽토그램(직업: 활동지원사)
   └─ 음성: "이 친구가 민지님 일기를 봐도 될까요?"
        ↓
[3] AAC 동의 (Yes/No 카드)
   └─ 큰 카드 2장: [동그라미 = 좋아요] [엑스 = 싫어요]
   └─ 한 번 더 확인: "정말로 보여줘도 될까요?" (실수 방지)
        ↓
[4] 동의 완료 — 축하 애니메이션
   └─ "고마워요! 민지님이 정해줬어요"
   └─ 음성으로 "엄마에게 알려드릴게요" 안내
        ↓
[5] 보호자(엄마)에게 푸시 알림 → "민지님이 직접 동의했어요"
```

#### 6.1.3 핵심 UX 원칙 (당사자 모드 전용)
- **1화면 1결정**: 동시에 2개 이상의 선택지를 묻지 않는다
- **2단계 확인**: 모든 권한·동의 행동은 "한 번 더 묻기" 필수
- **취소는 항상 가능**: 어떤 화면에서도 큰 [집] 버튼으로 처음 화면 복귀
- **부정적 단어 회피**: "삭제" 대신 "지울게요", "오류" 대신 "다시 해볼게요"
- **시간 압박 없음**: 자동 타임아웃 없음, 본인이 스스로 닫을 때까지 유지

| 단계 | 기존 페인 | 목표 First Value |
|------|----------|----------------|
| 가입 후 첫 5분 | "뭘 해야 할지 모르겠다" | 자녀 프로필 1개 완성 |
| 첫 30분 | "사진 5,000장을 어떻게 올리지?" | 사진 100장 자동 분류 결과 미리보기 |
| 첫 1시간 | "이게 정말 도움이 될까?" | 인계서 샘플 1장 PDF 다운로드 |
| 첫 1주 | "혼자 쓰는 느낌이다" | 학교 담임/활동지원사 1명 연결 완료 |

### 7.2 7단계 온보딩 마법사

```
[Step 1] 환영 + 본인인증 (PASS/카카오)
   └─ 진행 바: ●○○○○○○ (1/7)
        ↓
[Step 2] 자녀 등록 (이름/생년월일/장애 등급)
   └─ 자녀가 여러 명이면 "또 한 명 등록" 가능 (스킵 가능)
   └─ 진행 바: ●●○○○○○
        ↓
[Step 3] 현재 소속 기관 선택 (학교/복지관/없음)
   └─ 자동완성 + "직접 입력" 옵션
   └─ 추후 변경 가능 안내
        ↓
[Step 4] 기존 기록 마이그레이션 (선택)  ★ 핵심
   ┌─────────────────────────────────────┐
   │ 카톡 사진/영상   [업로드] (1만장까지) │
   │ 휴대폰 갤러리   [선택]              │
   │ 종이 노트       [사진 찍기→OCR]     │
   │ 엑셀/한글 파일  [드래그앤드롭]      │
   │ 건너뛰기        [나중에 할게요]     │
   └─────────────────────────────────────┘
   └─ 백그라운드 처리 + 진행률 (스킵 가능)
        ↓
[Step 5] 권한 모델 선택 (자녀 나이 기반 자동 추천)
   └─ 만 18세 미만: "보호자 전권" 자동
   └─ 만 18세 이상: "보호자 + 당사자 공동" 추천
   └─ 후견 결정문 있음: "후견 모드" 안내
        ↓
[Step 6] 알림 채널 설정 (푸시/SMS/이메일/카톡)
   └─ 권한 만료·인계서 수신·신규 기록 3종 기본 ON
        ↓
[Step 7] First Value 체험
   ├─ 미리 분류된 사진 타임라인 미리보기 (10장)
   ├─ AI 자동 생성 인계서 샘플 PDF 다운로드
   └─ "축하합니다! 레코다레 사용 준비 완료"
```

### 7.3 마이그레이션 마법사 상세 (Step 4)

#### 7.3.1 입력 채널별 처리

| 채널 | 처리 방식 | 시간 |
|------|----------|:----:|
| 카톡 사진 (.zip 백업) | 사진 EXIF 날짜 → 시기 자동 분류 + GPT-Vision으로 식사/활동/의료 자동 태깅 | 1만장 / 14일 |
| 휴대폰 갤러리 | 동일 처리 + 위치 정보 익명화 | 5천장 / 7일 |
| 종이 노트 사진 | OCR (네이버 클로바 OCR) → 텍스트 추출 + 날짜 추정 | 100장 / 3일 |
| 엑셀/한글 | 행 단위 파싱 (날짜/이벤트/메모 컬럼 자동 매핑) | 즉시 |

#### 7.3.2 마이그레이션 중 UX 약속
- **백그라운드 처리 + 푸시 알림**: 보호자는 앱 닫고 일상 가능
- **정확도 70% 보장 + 보호자 검수 UI**: AI가 분류한 결과를 보호자가 카드 스와이프로 검증
- **되돌리기 가능**: 잘못 분류된 사진 한 번에 재분류
- **개인정보 분리 동의**: 마이그레이션 전용 별도 동의서 (얼굴 인식 사용 여부 옵트인)

## §8. 보완 — 오프라인 연속성 (Offline-First 음성 기록)

### 8.1 시나리오 — 활동지원사 이수진의 산간 캠프

```
오전 10시 : 강원도 산간 발달장애인 캠프 도착 (4G 없음)

오후 06시 : 캠프 복귀, WiFi 연결됨
            → 앱이 자동으로 큐에 쌓인 5건 백그라운드 동기화
            → 충돌 감지 (동일 시간대 보호자 기록 1건 발견)
            → 사용자에게 병합/덮어쓰기 선택 UI 제공
```

| **저장 용량** | 1인당 최대 500MB (오프라인 음성 200건 ~ 2시간 분량) |
| **암호화** | 로컬 저장 시에도 AES-256 + 사용자 패스워드 기반 키 |

## §10. 보완 — 인지 접근성 (Cognitive Accessibility) 심화

### 10.1 v1.0 한계 진단

v1.0 NFR-05는 "색약/저시력"만 명시. 인지 접근성(쉬운말, 반복 확인, 실수 회복, 주의 분산 방지)은 미정의 → v2에서 보강.

### 10.2 인지 접근성 5대 원칙 (한국형 발달장애인 웹접근성 지침 + WCAG 2.1 Cognitive)

#### 원칙 1. 쉬운말 (Easy-to-Read) 모드 전역 지원

| 항목 | 기준 |
|------|------|
| 어휘 | 초등 2학년 수준 어휘 (한국어 기초 1,500어휘 내) |
| 문장 길이 | 최대 15음절 |
| 한자/외래어 | 0건 (모두 한글 풀어쓰기) |
| 픽토그램 동반 | 모든 핵심 동작 버튼은 픽토그램 + 글자 병기 |
| 음성 안내 | 모든 텍스트는 TTS로 들을 수 있음 |

**예시**:
| 일반 모드 | 쉬운말 모드 |
|----------|-----------|
| "권한 이양에 동의하시겠습니까?" | "선생님이 내 일기를 볼 수 있게 해줄까요? (그림 + 음성)" |
| "민감정보 처리 동의" | "내 비밀 이야기를 보여줄게요. 괜찮을까요?" |
| "오류가 발생했습니다" | "다시 한 번 해볼게요" |

#### 원칙 2. 반복 확인 UI (Double-Confirm)
- 모든 비가역 행동(삭제·권한 해지·동의·전송)은 2단계 확인 필수
- 1단계: 행동 카드 탭
- 2단계: "정말로 할까요? — [네/아니오]" 모달 (3초 후 자동 닫힘 없음)

#### 원칙 3. 실수 회복 (Undo) 항상 노출
- 모든 화면 좌상단 [되돌리기] 큰 버튼
- 최근 10개 행동까지 되돌리기 가능
- 영구 삭제도 30일 휴지통 보존

#### 원칙 4. 주의 분산 방지
- 동시에 떠 있는 모달/팝업 1개만 허용
- 자동재생 영상 금지, 깜박이는 광고/배너 0개
- 한 화면에 CTA(주요 행동 버튼) 1~2개만

#### 원칙 5. 시간 압박 제거
- 자동 로그아웃 시간 30분 (일반 5분의 6배) — 당사자 모드 시
- 폼 입력 자동 저장 (Draft) 매 키스트로크
- "X초 안에 답하세요" 타이머 일체 금지

### 10.3 인지 접근성 검증 도구
- **사용성 테스트**: 발달장애 당사자 자문단 5명 분기별 평가
- **자동 검사**: WCAG 2.1 AA + 한국형 웹접근성 지침 + 자체 "쉬운말 어휘 검사기"
- **인증**: 한국웹접근성평가센터 인증 + 보건복지부 정보접근성 인증

### 10.4 신규 FR/NFR (인지 접근성)

| FR/NFR# | 요구사항 | 우선순위 |
|---------|---------|---------|
| FR-40 | 쉬운말 모드 전역 토글 (어휘/문장/픽토그램/TTS 동시 적용) | P0 |
| FR-41 | 모든 비가역 행동에 2단계 확인 UI 의무 | P0 |
| FR-42 | 30일 휴지통 + 10개 행동 되돌리기 | P1 |
| NFR-09 | 한국웹접근성 인증 + 보건복지부 정보접근성 인증 18개월 내 취득 | P0 |
| NFR-10 | 발달장애 당사자 자문단 5인 분기별 사용성 평가 의무 | P1 |

### 12.4 v2.1 Pre-mortem 추가 리스크 (Risk #7~#9)

#### Risk #4. 신규 보호자가 마이그레이션 중 이탈 (v2.0)
**가상**: 보호자가 카톡 사진 1만장 업로드 후 AI 분류 결과의 오류를 보고 "AI 못 믿겠다" 이탈.
**대응**: First Value를 사진 마이그레이션 이전에 배치 (Step 3까지 완주 시 인계서 샘플 즉시 제공), 분류 정확도 70% 보장 명시 + 보호자 직접 정정 UI 강화.

#### Risk #5. 알림 피로로 인한 무음 처리 → 권한 만료 사고 (v2.0)
**가상**: 보호자가 일지 알림 폭주로 전체 알림 OFF → 권한 만료 D-1 알림도 미수신 → 활동지원사 데이터 접근 차단으로 입소 일정 차질.
**대응**: 알림 유형별 분리 토글 강제 (전체 OFF 옵션 없음, "중요 알림은 항상 ON" 가이드), 만료 D-1은 SMS+카톡 다중 채널 필수.

#### Risk #6. 시설장 일괄 권한 갱신 실수 (v2.0)
**가상**: 시설장이 5명 활동지원사 권한을 6개월로 갱신해야 하는데 실수로 1개월 선택 → 한 달 후 일괄 만료.
**대응**: 일괄 갱신 전 "영향 범위 + 갱신 결과 만료일" 명확 미리보기, 시설장 행동 30일 내 1회 일괄 되돌리기 가능.

#### **Risk #7. 18세 권한 이양 절차 오류 → 자립 사고 (v2.1, 확률 중간 / 영향 긴급)**
**가상**: D-0 도래 시 권한 모델 자동 전환이 실패해 자녀가 자기 데이터에 접근 못 함 → 의료 응급 상황에서 보호자 권한도 만료 → 응급 의료 결정 공백.
**대응**: FR-52에 **D+30 정착 점검 + 응급 권한 잔존 (보호자 의료 알림 수신권 30일 유지)** 명시. D-0 자동 전환 전 D-7부터 "최종 확인 모달" 다중 채널. 전환 실패 시 자동 롤백 + 법무 자문 핫라인.

#### **Risk #8. B2G 청구 오류로 시설 운영 손실 (v2.1, 확률 중간 / 영향 높음)**
**가상**: FR-56의 분기 청구서가 잘못된 이용자를 대상으로 생성되어 시설장이 검토 없이 제출 → 보건복지부 환수 통보 + 시설 신뢰도 타격.
**대응**: 청구서 발행 전 **시설장 검토 모달 의무화** (자동 발행 금지), 청구 대상자 미리보기 + 보호자 동의 상태 자동 점검 + 환수 발생 시 영업팀 자동 알림 + 분기 1회 PoC로 정확도 95%+ 도달 시까지 수동 청구 병행.

#### **Risk #9. 후견 결정문 검증 실패로 권한 부정 사용 (v2.1, 확률 낮음 / 영향 긴급)**
**가상**: 대법원 가족관계등록 검증 API가 일시 중단되어 FR-59의 자동 검증을 통과한 위조 결정문이 권한 부여로 이어짐 → 본인 의사에 반하는 의료·재산 결정.
**대응**: 검증 실패 시 **기본값 = 일반 보호자 권한** (후견 권한 미활성), 검증 성공 후에도 **D+7 보호자(가족)·당사자 모두에게 후견 등록 알림 발송** + 이의 제기 30일 채널 + 분기 샘플링 수동 재검증.

#### **Risk #10. 자가진단을 정식 의료 진단으로 오인 (v2.2, 확률 중간 / 영향 긴급)**
**가상**: 보호자가 K-DST 결과를 의료적 진단서로 오해해 자녀 어린이집·학교에 제출 → 무자격 의료 진단 유통 → 의료법 §27 위반 분쟁 + 시설장 신뢰 손상.
**대응**: FR-68 면책 표기를 모든 결과 화면·PDF에 의무화 (자동 워터마크), "이 문서를 의료적 진단 용도로 사용할 수 없습니다" PDF 모든 페이지 표시, 보건복지부 의료법 자문 위촉(M3 이전), 분기 1회 LLM 출력 감사.

#### **Risk #11. AI 추이 분석 오해석으로 부적절한 의사결정 (v2.2, 확률 중간 / 영향 높음)**
**가상**: AI가 단일 도메인 변동(우연한 응답 누락)을 "급격한 하락"으로 잘못 해석 → 보호자가 자녀를 부적절한 치료에 노출.
**대응**: 추이 분석은 **최소 3회 이상 데이터 누적 시에만 활성화** (2회 이하는 "데이터 부족" 안내), 인사이트는 항상 "전문가 상담 권장" 문구 동반, 보호자 피드백 분기 만족도 측정.

#### **Risk #12. 정부 정책 변화로 B2G 청구 모델 무효화 (v2.3, 확률 중간 / 영향 긴급)**
**가상**: 발달장애인 권리보장법 개정 또는 발달재활서비스 바우처 정책 폐지·축소 시 B2G 청구 매출 (32억/3년) 가설 무효화. 또는 디지털 인계서가 법정 양식으로 인정받지 못해 종이 양식 병용 강제.
**대응**:
- 보건복지부 정책 모니터링 위원 위촉 (분기 1회 정책 변화 보고)
- 본 PRD에 시나리오 매트릭스 추가 (낙관·중립·비관 3 시나리오별 대응 액션)
- B2C (Personal Plus) + B2B (Facility) 매출 비중을 50:30:20 (B2C·B2B·B2G)로 분산 — 단일 의존성 해소
- 디지털 인계서가 법정 양식 미인정 시 보호자 다운로드 + 종이 양식 자동 변환 옵션 제공

## §15. v2.1 신규 — 18세 권한 이양 (Adulthood Transition)

### 15.1 배경

v1.0/v2.0은 "권한 이행"을 언급했으나 **만 18세 시점의 단계적 이양 절차에 대한 명시 FR이 없었음**. workflows §B12/§A1.4의 6개월 가이드를 정식 FR로 흡수.

> **WHY 이게 중요한가**: 보건복지부 자립지원 통계상, 권한 이양 절차가 없는 가정의 자녀는 만 19세 이후 시설 재진입율이 35% 더 높다. 이 단계는 자립 성공의 단일 최대 분기점.

## §16. v2.1 신규 — 케이스 회의 자료 자동 생성

### 16.1 배경

v1.0/v2.0의 U4(사회복지사·특수교사)는 "데이터 조회"만 정의되었고 **실제 업무 산출물(사례관리 회의 자료)의 자동 생성 흐름이 누락**. workflows §B17의 흐름을 FR로 흡수.

> **WHY 이게 중요한가**: 복지관 사회복지사는 케이스당 분기 1회 회의 자료 작성에 평균 60분 소요. 30케이스 담당 시 분기당 30시간 = 한 주 풀타임 노동. 이를 3분으로 줄이면 본업(상담)에 집중 가능.

### 16.2 워크플로우

```
[1] U4 "케이스 회의 자료" 진입 → [2] 대상 케이스 선택 (담당 케이스 리스트)
   → [3] 기간 + 4영역(교육/의료/돌봄/자립) 자동 추출 (권한 범위 마스킹 적용)
   → [4] AI 요약 — 4영역 각 3문장 + 핵심 사건 5건 + 의료/위험 알림 (3페이지 분량)
   → [5] U4 검토 + 편집 (수정 권한 = 본인 작성 메모만, AI 추출 사실은 편집 불가)
   → [6] PDF 생성 + 회의 참석자 안전 공유 (만료 링크, 7일 + 비밀번호 옵션)
   → [7] 감사 로그 5년 보존 (FR-47 연동) + 보호자에게 회의 자료 생성 알림
```

### 16.3 신규 FR

| FR# | 요구사항 | 우선순위 | 마일스톤 |
|-----|---------|---------|---------|
| **FR-53** | 케이스 회의 자료 AI 자동 생성 (4영역 추출 + 3페이지 PDF + 만료 링크 공유, 권한 마스킹 자동 적용) | P0 | GA 1.0 (M7-M12) |

### 16.4 제약·정책

- **권한 마스킹**: U4의 권한 범위 밖 데이터는 자료에서 자동 제외 + "권한 외 데이터 N건 비공개" 표시
- **편집 제약**: AI가 추출한 사실(일지 인용)은 편집 불가, U4의 해석·소견 메모만 편집 가능 → 사실-해석 분리
- **보호자 사전 검토 옵션**: 보호자가 옵션 ON 시 회의 자료 발행 24h 전 보호자 검토 단계 추가

### 16.5 측정 KPI

| KPI | 목표 |
|-----|------|
| 자료 작성 시간 (60분 → X분) | **3분 이하** |
| 자료 정확도 (사실 오류율) | < 5% (분기 샘플링) |
| 보호자 이의 제기율 (자료 내용 항의) | < 2% |

## §20. v2.2 신규 — 자가진단 도구 + AI 추이 분석 (의료 진단 아님)

### 20.1 배경

PRD v2.1까지 61개 FR이 정의되었으나 **신규 진단 보호자(영유아 페르소나)의 진단 전·후 모니터링 도구가 0건**. 발달지연 의심 시기에 보호자가 의존할 수 있는 객관적 자가진단 도구가 없어, "전문의 진단 전 막연한 불안"의 페인을 해소하지 못한다.

> **핵심 포지셔닝 (의료법 안전)**: 본 도구는 **의료적 진단이 아니라 "보호자 관찰 의견의 시간적 추이 기록"**이다. 모든 결과 화면·PDF·알림에 "정식 진단은 전문의 상담 필요"를 의무 표기하여 의료법·약사법 위반 분쟁을 사전 차단한다.

### 20.4 AI 추이 분석 — FR-67 명세

#### 입력
- 동일 사용자(자녀)의 동일 도구 자가진단 결과 시계열 N회 (최소 2회 이상)

#### 처리
1. **도메인별 점수 추이 시각화**: 라인 차트 (각 도메인 N회 기록)
2. **추세 분류**: ↑상승(+2점 이상) / →정체(±1점) / ↓하락(-2점 이상)
3. **자연어 인사이트 생성** (LLM): 변화 패턴 + 보호자 권장 행동 (의학적 권고는 제외)
4. **이상치 알림**: 단일 도메인이 +3점 이상 급변 또는 -3점 이상 급하락 시 보호자 알림 (전문의 상담 안내 포함)

#### 출력 예시
```
박지훈 — 자기결정능력 자가체크 추이 (3회: 2024-12 → 2025-12)
─────────────────────────────────────────────────────
자율성       12 → 15 → 18  (+6점, 상승 ↑)
자기조절     14 → 16 → 19  (+5점, 상승 ↑)
심리적 권한  15 → 18 → 19  (+4점, 상승 ↑)
자기인식     17 → 19 → 20  (+3점, 상승 ↑)
─────────────────────────────────────────────────────
🤖 AI 인사이트: 전 영역 +3~6점 상승. AAC 동의 활동이 자기결정 능력
   향상에 긍정적 영향. 18세 이양 D-90 단계에서 의료 항목 부분 이양 진행에 적합.
   ⚠ 정식 평가는 발달재활서비스 전문가 상담 권장.
```

#### 화면 위치
- `/parent/screening` — 자가진단 목록 + 다음 검사 예약
- `/parent/screening/{tool-key}` — 도구별 응답 화면 (K-DST 30~60문항)
- `/parent/screening/trend/{tool-key}` — AI 추이 분석 화면 (FR-67)
- `/case/screening/{personId}` — 사회복지사용 케이스별 추이 열람 (권한 마스킹 적용)

### 20.6 Risk 추가 — Pre-mortem

#### Risk #10. 자가진단을 정식 의료 진단으로 오인 (확률 中 / 영향 致命 → 긴급)
**가상**: 보호자가 K-DST 결과를 의료적 진단서로 오해해 자녀 어린이집·학교에 제출 → 무자격 의료 진단 유통 → 의료법 §27 위반 분쟁 + 시설장 신뢰 손상
**대응**:
- FR-68 면책 표기를 모든 결과 화면·PDF에 의무화 (자동 워터마크)
- "이 문서를 의료적 진단 용도로 사용할 수 없습니다" — PDF 모든 페이지 표시
- 보건복지부 의료법 자문 위촉 (M3 이전)
- 분기 1회 LLM 출력 감사 (의료적 표현 회피 검증)

#### Risk #11. AI 추이 분석 오류로 인한 부적절한 의사결정 (확률 中 / 영향 高)
**가상**: AI가 단일 도메인 변동(예: 우연한 응답 누락)을 "급격한 하락"으로 잘못 해석 → 보호자가 자녀를 부적절한 치료에 노출
**대응**:
- 추이 분석은 최소 3회 이상 데이터 누적 시에만 활성화 (2회 이하는 "데이터 부족" 안내)
- 인사이트는 항상 "전문가 상담 권장" 문구 동반
- 보호자 피드백 수집 — "AI 인사이트가 도움이 됐나요?" 분기별 만족도 측정

## §24. v2.6 신규 — PBS(긍정적 행동지원) 전생애 공통 모듈

> 기준 문서: `docs/00-pm/record_data/2 기록종류별 세부기록 항목.md` §5 + `record_data/3 전생애 기록 체계.md` §3.1  
> PBS(Positive Behavior Support)는 영유아기부터 고령기까지 전 생애에 걸쳐 발생하는 도전행동을 과학적으로 기록·분석하는 공통 모듈이다.

### 24.1 PBS 개요

도전행동(자해·타해·기물파손·소리지르기 등)은 장애 당사자의 **말이 아닌 행동으로 표현하는 의사소통**이다. Recordare는 ABC 분석법(선행사건-행동-결과)을 정형 데이터로 수집하여 AI 패턴 분석의 기반으로 삼는다.

```
[선행 사건 (Antecedent)] → [도전행동 (Behavior)] → [결과 및 대처 (Consequence)]
   - 소음·빛·일정 변경         - 자해/타해/기물파손       - 안정실 이동
   - 요구 거절                 - 소리지르기/바닥 눕기     - AAC 판 제공
   - 신체 불편감(치통 등)       - 부적절한 행동            - 노이즈캔슬링 적용
```

### 24.2 PBS 입력 데이터 스키마

| 필드 | 데이터 타입 | 필수 | 설명 |
|------|-----------|:---:|------|
| `occurredAt` | Timestamp | ✅ | 시스템 자동 태깅 (yyyy-MM-dd HH:mm:ss) |
| `location` | 선택형 | ✅ | 학교 교실 / 가정 거실 / 대중교통 / 작업장 / 기타 |
| `antecedent.type` | 다중 선택 | ✅ | 감각(소음·빛·촉각) / 환경(일정변경·낯선 사람) / 요구(하기 싫은 과제·거절) |
| `antecedent.detail` | 텍스트 | — | "갑작스러운 경적 소리", "수업 내용 변경" 등 자유 서술 |
| `behavior.type` | 다중 선택 | ✅ | 자해 / 타해 / 기물파손 / 소리지르기 / 바닥 눕기 / 부적절한 성적 행동 |
| `behavior.intensity` | 1~5 척도 | ✅ | 1(경미) ~ 5(응급 수준) |
| `behavior.durationMin` | 정수 (분) | ✅ | 지속 시간 |
| `consequence.action` | 다중 선택 | ✅ | 안정실 이동 / AAC 판 제공 / 신체 보정법 / 진정제 복용 / 간식 지급 |
| `consequence.detail` | 텍스트 | — | 중재자 관찰 의견 자유 서술 |
| `contextData` | JSON | — | AI 연계용 — 날씨, 전날 수면 시간, 식사량, 복약 여부 |
| `recordedBy` | 역할 코드 | ✅ | SW / SE / MP / VC / CG |

### 24.3 AI 패턴 분석 (FR-85)

PBS 일지가 누적(최소 10건)되면 LLM 에이전트가 다음을 자동 분석한다:

- **빈도 추이**: 주간/월간 도전행동 발생 빈도 그래프
- **선행사건 패턴**: "오후 2~3시 + 소음 + 과제 변경" 조합 시 발생율 80% 등 패턴 추론
- **컨텍스트 상관**: 수면 부족 다음 날 강도 척도 +1.5 상승 등 외부 변수 연계
- **중재 효과**: 특정 대처법(안정실 이동 vs AAC 판) 효과성 비교

## §25. v2.6 신규 — PCP(개인 중심 계획) 성인기 레코드 체계

> 기준 문서: `docs/00-pm/record_data/3 전생애 기록 체계.md` §3단계 3.1  
> PCP(Person-Centered Planning)는 당사자의 강점·선호·꿈을 중심에 두고 서비스를 설계하는 국제 표준 접근법.

### 25.1 PCP 개요

| 항목 | 내용 |
|------|------|
| **목적** | 장애 결함 중심이 아닌 **당사자 강점·선호 중심** 삶의 계획 |
| **주 기록자** | 당사자(SR) + 보호자(CG) 공동 — 당사자가 1차, 보호자가 검토 |
| **적용 생애주기** | 전환기(15~24세) + 성인기(25~64세) |
| **Recordare 활용** | 매칭 알고리즘 입력값 — 당사자 선호 기반 서비스/모임 매칭 |

### 25.2 PCP 입력 데이터 스키마

| 영역 | 질문 (쉬운 말 모드) | 데이터 타입 | 예시 |
|------|-------------------|-----------|------|
| **좋아하는 것** | "내가 제일 좋아하는 활동은요?" | 다중 선택 + 텍스트 | 음악 감상, 그림 그리기, 요리 |
| **싫어하는 것** | "내가 힘들어하는 상황은요?" | 다중 선택 + 텍스트 | 소음이 큰 곳, 갑작스러운 일정 변경 |
| **꿈·목표** | "앞으로 내가 하고 싶은 것은요?" | 텍스트 (AAC 픽토그램 선택 가능) | 혼자 살기, 바리스타 되기 |
| **의사소통 방식** | "나는 주로 어떻게 이야기해요?" | 선택형 | 구어 / AAC / 신체 표현 / 행동 징후 |
| **자립 우선순위** | "가장 먼저 혼자 할 수 있고 싶은 것은?" | 순위 선택 (최대 3개) | 1순위: 대중교통 혼자 타기 |
| **지원 필요 영역** | "지금 가장 도움이 필요한 것은요?" | 다중 선택 | 금전 관리, 요리, 직장 찾기 |

## §26. v2.6 신규 — 사후 지원기 기록 체계 (주보호자 유고 대비)

> 기준 문서: `docs/00-pm/record_data/3 전생애 기록 체계.md` §5단계  
> 지적장애인의 주 돌봄자(부모)가 사망하거나 돌봄 능력을 상실했을 때를 대비하는 사전 계획 및 전환 기록.

### 26.1 사후 지원기 핵심 데이터

| 구분 | 항목 | 기록 주체 | 설명 |
|------|------|---------|------|
| **SNT 신탁** | 특별부양신탁 가입 여부, 수탁 기관, 월 지급 설계액, 비상 인출 권한 대리인 | CG, LG | 발달장애인 특별부양신탁 (은행/공공재단) — 부모 사후 당사자 생활비 자동 집행 |
| **후견 정보** | 후견인 형태(특정/성년/공공), 성명·연락처, 가정법원 보고서 아카이브 | LG | 정기 제출 보고서 PDF 저장 + 승인 완료 목록 관리 |
| **거주지 전환** | 단기 보호시설 예약 상태, 장기 입소 예정 시설 리스트, 전환 발동 조건 | CG, SW | 부모 사망/입원 즉시 이송할 시설 사전 예약 + 연락처 |
| **애도 케어** | 주보호자 사망 후 당사자 심리 반응 기록, 행동 중재 가이드 | SW, CG | 우울/불안 징후, 비상 완충 활동 적용 이력 |

### 26.2 사후 지원 워크플로우

```
[주보호자 유고 발생]
       ↓
[Step 1] 거주지 전환 발동
   └─ 사전 예약 단기 보호시설 자동 알림 발송 (2순위 돌봄자, 사회복지사)
       ↓
[Step 2] SNT 신탁 집행 시작
   └─ 수탁 기관에 집행 개시 알림 → 월 지급액 당사자 계좌 자동 이체
       ↓
[Step 3] 후견인 권한 이행
   └─ LG가 주요 의사결정 대리 + 가정법원 보고 시작
       ↓
[Step 4] 당사자 심리 지원
   └─ SW가 PBS 집중 모니터링 + 애도 반응 기록 + 완충 활동 가이드 적용
```
