## Executive Summary

| 관점 | 내용 |
|------|------|
| **Problem (문제)** | 지적장애인 당사자의 생애주기별 기록(영유아 → 학령기 → 전환기 → 성인기 → 고령기)이 기관 전환 시마다 단절되어, 매번 신규 이해관계자가 당사자를 처음부터 파악해야 하며, 성인이 된 후에도 보호자 중심으로 정보 주권이 고착되어 자기결정권이 배제됨. |
| **Solution (해법)** | (1) 당사자 중심 통합 타임라인, (2) AI STT + 자동 카테고리 태깅으로 현장 기록 부담 최소화, (3) 법률 기반 하이브리드 권한 매칭(보호자→당사자 이행), (4) 3분 마스터 인계서 자동 생성. |
| **Function/UX Effect** | 현장 전문가 일지 작성 시간 70% 단축, 기관 인계 시간 평균 2주 → 3분, 성인 당사자의 정보 동의 직접 행사율 0% → 60% (3년 내). |
| **Core Value** | "기록은 끊기지 않고, 권한은 당사자에게 자란다" — 지적장애인 평생 디지털 기록 자산화 및 자기결정권 회복. |

## Context Anchor

| 차원 | 내용 |
|------|------|
| **WHY** | 국내 발달장애인 약 26.2만 명(2024 보건복지부 통계 기준), 가족 포함 시 78만 명이 기록 단절 / 인계 실패로 인한 케어 품질 저하를 경험. 자기결정권 침해는 UN장애인권리협약 제12조 위배 소지. |
| **WHO** | (P) 당사자 (15~40세, AAC/쉬운말 필요), (B) 보호자 (40~70세, 행정 피로 누적), (E) 현장 전문가 (특수교사·활동지원사·사회복지사, 이직률 높음). |
| **RISK** | (1) 민감정보 처리 위반 (개인정보보호법 §23), (2) 성년후견 권한 매핑 오류 (민법 §9), (3) AAC 표준 미준수로 인한 인지적 접근성 실패, (4) 권한 이행 시점의 법적 분쟁. |
| **SUCCESS** | 18개월 내 활동지원사 일지 평균 작성시간 8분 → 2분, 비콘 사용자(중증 발달장애인 거주시설 5곳) 인계서 작성률 80%, NPS 30+. |
| **SCOPE** | MVP: 보호자 + 활동지원사 양면 + AI 음성기록 + 인계서 + 미성년 권한 모델. 제외: 후견인 자동판정, IoT 통합, 의료 PHR 연동(v2). |

## 2. Strategy — 가치 제안 및 비즈니스 모델

### 2.1 Value Proposition (JTBD 6-Part Framework, 이해관계자별)

#### 2.1.1 당사자 (Person with Intellectual Disability, 17세+)

| 6-Part | 내용 |
|--------|------|
| **When** (상황) | 성인이 되어 새로운 복지관/직장에 가는 날, 누군가 내 정보를 보여달라고 할 때 |
| **I want to** (Job) | 내 정보를 누구에게 보여줄지 직접 정하고 싶다 |
| **So I can** (목표) | 어른답게 내 일을 내가 결정한다 |
| **Help me** (요청) | 어려운 글자 말고, 그림과 음성으로 알려줘 |
| **Avoid** (회피) | 엄마가 모든 걸 대신 결정하는 것 |
| **Make me feel** (감정) | 내가 존중받고 있다고 느끼고 싶다 |

#### 2.1.2 보호자 (Caregiver, 40~70세)

| 6-Part | 내용 |
|--------|------|
| **When** | 자녀가 새로운 기관·학교에 가거나, 자녀가 성인이 되어 권한이 이행될 때 |
| **I want to** | 자녀의 발달·돌봄 이력을 한 번에 인계하고, 권한을 안전하게 단계적으로 넘기고 싶다 |
| **So I can** | 자녀가 어디서든 동일한 케어 품질을 받고, 내가 없을 미래를 준비한다 |
| **Help me** | 법적으로 안전한 권한 이양 절차를 자동으로 안내해줘 |
| **Avoid** | 매번 30페이지짜리 인계서를 다시 쓰고, 권한 이양 시 법적 분쟁이 생기는 것 |
| **Make me feel** | "이제는 시스템이 도와준다"는 안심 |

#### 2.1.3 활동지원사·특수교사 (Field Professional)

| 6-Part | 내용 |
|--------|------|
| **When** | 하루 8시간 돌봄 후, 행정 일지를 작성해야 할 때 / 새로운 이용자를 처음 만났을 때 |
| **I want to** | 음성으로 1분 안에 일지를 끝내고, 신규 이용자의 핵심 주의사항을 3분 안에 파악하고 싶다 |
| **So I can** | 행정에 뺏긴 시간을 실제 돌봄에 쓰고, 신규 이용자에게 즉시 적절한 케어를 제공한다 |
| **Help me** | 음성을 자동으로 카테고리 분류해주고, 인계서를 요약해줘 |
| **Avoid** | 야근으로 일지를 쓰는 것 / 첫 주에 사고가 나는 것 |
| **Make me feel** | 전문성이 존중받는다고 느끼고 싶다 |

#### 2.1.4 사회복지사 (Social Worker / Coordinator)

| 6-Part | 내용 |
|--------|------|
| **When** | 사례관리 회의 / 외부 기관과 케이스 협업 시 |
| **I want to** | 당사자의 다영역(교육·의료·돌봄·자립) 기록을 한눈에 보고 케이스 플랜을 짜고 싶다 |
| **So I can** | 데이터 기반 사례관리로 중복지원·지원공백을 줄인다 |
| **Help me** | 권한 범위 안의 데이터만 안전하게 조회·공유하게 해줘 |
| **Avoid** | 보호자에게 같은 정보를 또 묻는 상황 |
| **Make me feel** | 전문가로서 영향력 있는 결정을 내린다고 느끼고 싶다 |

### 2.3 SWOT 분석

| | 도움 (Helpful) | 방해 (Harmful) |
|---|----------------|----------------|
| **내부 (Internal)** | **Strengths**: 사회적 임팩트 명확, 정책 부합(권리보장법 2024 강화), 경쟁사 부재 영역(권한 이행 + 평생기록), AI STT 기술 성숙 | **Weaknesses**: 초기 사용자 확보 비용 높음(B2B 영업 사이클 3~6개월), 한국어 비표준 발화 데이터셋 부족, 도메인 전문 PM·디자이너 확보 어려움 |

## 5. PRD — Product Requirements Document (8 Sections)

### 5.1 Problem Statement

지적장애인 당사자의 평생 기록은 영유아 → 학령기 → 전환기 → 성인기 → 고령기로 이어지지만, 현재는 **기관 이동마다 단절**되고, **현장 전문가의 행정 부담**으로 적시·정확한 기록이 어렵고, **성인 후에도 보호자 중심으로 정보 주권이 고착**되어 자기결정권이 배제된다. 이는 케어 품질 저하, 행정 비효율, UN장애인권리협약 위배 소지를 야기한다.

### 5.2 Goals & Success Metrics

### 5.3 User Stories (INVEST 체크 완료)

#### Epic 1. 통합 타임라인
- **US-01**: 보호자로서, 자녀의 영유아~현재까지의 모든 기록을 단일 타임라인에서 보고 싶다. 그래야 자녀의 발달 추이를 한눈에 파악할 수 있다.
- **US-02**: 활동지원사로서, 신규 이용자 첫 만남 전에 3분 분량의 핵심 주의사항 요약을 받고 싶다. 그래야 첫 주 사고를 예방할 수 있다.
- **US-03**: 사회복지사로서, 케이스의 교육/의료/돌봄/자립 4영역 이력을 한 화면에서 보고 싶다. 그래야 케이스 플랜을 효율적으로 작성할 수 있다.

#### Epic 2. AI 간편 기록
- **US-04**: 활동지원사로서, "오늘 철수 식사 잘함, 약간 불안" 을 음성으로 말하면 식사·행동·정서 카테고리로 자동 분류·저장되길 원한다. 그래야 행정 시간을 줄일 수 있다.
- **US-05**: 특수교사로서, 학급 전체 학생의 공통 활동 기록을 1번 입력으로 전 학생 일지에 반영하고 싶다.
- **US-06**: 보호자로서, 카톡에 흩어진 사진 수천 장을 드래그앤드롭으로 일괄 업로드하면 AI가 시기/카테고리로 자동 정리해주길 원한다.

#### Epic 3. 권한 이행
- **US-07**: 당사자(17세)로서, "내 정보를 OO 선생님에게 보여줄까요?" 라는 질문에 그림 Yes/No 카드로 답하고 싶다.
- **US-08**: 보호자로서, 자녀가 18세가 되었을 때 권한 이행 단계별 가이드와 시뮬레이션을 받고 싶다. 그래야 안전하게 이양할 수 있다.
- **US-09**: 보호자로서, 자녀가 성년후견을 받은 경우 후견인 합의결제 UI로 함께 동의할 수 있어야 한다.
- **US-10**: 시설장으로서, 계약 활동지원사의 권한이 계약 만료일에 자동 회수되길 원한다.

#### Epic 4. 인계서
- **US-11**: 보호자로서, 신규 기관에 보낼 인계서를 자동 생성하고 PDF로 출력하고 싶다.
- **US-12**: 신규 기관 담당자로서, 인계서 수령 시 권한 이양 워크플로(현 기관장 승인)를 시스템에서 처리하고 싶다.

#### Job Stories (보완)
- **JS-01**: 자녀의 첫 그룹홈 입소일을 앞두고 있을 때, 보호자가 그룹홈 새 사회복지사에게 자녀의 식이/투약/돌발행동 트리거를 사전 공유할 수 있도록 도와줘. 그래야 입소 첫날부터 안전한 케어가 시작된다.
- **JS-02**: 활동지원 계약이 갱신되지 않은 날, 시스템이 해당 지원사의 모든 데이터 접근권을 자동 회수하도록 도와줘. 그래야 개인정보 유출이 차단된다.

### 5.7 Pre-mortem (Top 3 Risks)

#### Risk #1. STT 한국어 비표준 발화 정확도 미달 (확률 높음 / 영향 높음)
**가상 실패 시나리오**: 출시 3개월 후, 활동지원사들이 "받아쓰기가 너무 틀려서 다시 입력해야 한다"는 불만 폭증 → MAU 50% 이탈.
**대응**:
- MVP 전 30일 데이터 수집 PoC (E1) 필수
- Naver Clova Speech + 자체 파인튜닝 + 사용자 교정 학습 루프
- Fallback: 텍스트 입력 모드 항상 노출, 후보군 3개 보여주기

#### Risk #2. 개인정보보호법 위반 사고 (확률 중간 / 영향 긴급)
**가상 실패 시나리오**: 1년차에 활동지원사 계정 탈취 → 100명의 발달장애 일지 유출 → 과징금 + 브랜드 타격.
**대응**:
- ISMS-P 인증 18개월 내, 모든 민감정보 컬럼 암호화
- 활동지원사 2FA 의무화, 이상 접근 패턴 탐지
- 사이버 보험 의무 가입, 침해 대응 IR 플레이북 사전 작성

#### Risk #3. 권한 이양 시점의 보호자 저항 (확률 높음 / 영향 중간)
**가상 실패 시나리오**: 자녀 18세 시 권한 이행 알림에 보호자가 "내 자식 데이터를 왜 못 보냐"며 분노 → 이탈/부정 리뷰.
**대응**:
- 권한 "이행"이 아닌 "공동 사용" 모델로 framing (보호자 권한 유지 + 당사자 권한 추가)
- 변호사 1회 무료 상담 쿠폰 + 부모 커뮤니티 교육 콘텐츠
- 단계적 학습형 시뮬레이션 (15세부터 5년에 걸쳐 자연스럽게)
